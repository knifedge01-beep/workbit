# Runbooks

Operational procedures: deploy, rollback, incident response, and recurring tasks.

## Suggested runbooks

- **deploy.md** — Build, env, and deploy steps for frontend and API.
- **rollback.md** — How to roll back a release (revert tag, redeploy previous, DB if needed).
- **incident.md** — Who to check, where logs live (Logbit), how to disable a feature or scale.

Add new runbooks as markdown files and link them from this README.
