# Architecture decision records (ADRs)

This folder holds short, dated records for important engineering decisions (tech choices, API shape, security, etc.).

## Format

One file per decision, e.g. `YYYY-MM-DD-short-title.md`:

- **Context** — What problem or option we were deciding.
- **Decision** — What we chose.
- **Consequences** — Trade-offs, follow-ups, what to revisit.
